# idea
# IDEA LOGGING COMMAND
When I use the command "/idea" followed by a description of an idea, you must perform the following actions:

1. **Read Context:** Understand the idea description provided by the user (what they want to document).
2. **Read File:** Read the current content of `diploma.md` to see existing ideas and avoid duplicates.
3. **Add Idea:** Add a new subsection under "## 💡 Идеи на будущее" in `diploma.md`, placing it after existing ideas (before the separator `---`).
4. **Format:** Use the following format for the new idea:

### [Название идеи (краткое, понятное)]

**Концепция**: [1-2 предложения о сути идеи]

**Детали реализации**:
- [Конкретные технические детали, как это можно реализовать]
- [Какие компоненты/модули затронуты]
- [Какие технологии/библиотеки могут понадобиться]

**Преимущества**:
- [Почему это полезно для проекта]
- [Какие проблемы это решает]
- [Как это улучшит пользовательский опыт или техническую сложность]

**Потенциальные сложности**:
- [Что может быть сложно в реализации]
- [Какие ограничения нужно учесть]

**Связь с текущим проектом**:
- [Как это связано с существующим функционалом]
- [Какие модули/сервисы будут затронуты]

---

**Tone:** Technical, clear, structured. Focus on explaining the idea comprehensively but concisely. Use bullet points for readability.
